# 第二轮界面升级设计文档评审

日期：2026-07-13  
评审对象：`docs/2026-07-13-第二轮界面升级设计文档.md`  
证据：四张实施后截图、当前 SwiftUI 实现（`WorkspaceView`、`ContentView`、`PairingView`、`RemoteAccessView`、`DiagnosticsSheet`、`PageContainer`）及第一轮 UX 基线。

## 结论

**有条件采纳，作为第二轮视觉收口的输入，而非原样执行清单。**

文档正确识别出第一轮的结构已完成、但工作站仍显稀薄，设备、连接和诊断的视觉完成度不足。它把“视觉分组”几乎等同于“为每段加卡片”、又准备把两个常用入口都升为主按钮，这会破坏第一轮已确立的原则：一个明确主动作、少量色彩、避免 dashboard 卡片拼贴。

正确方向是：**以宽屏自适应构图、行级层次和一张真正承担交互的状态面解决空洞；其余区域保持原生、克制的分组列表。** 完成本文 P0 修订后可进入编码。本轮不改变运行链路、配对状态机、Relay 默认语义或连接 Sheet 的单一目的地。

## 已核对的事实

- `WorkspaceView` 仍使用 `PageContainer` 的 880pt 默认工作列；`LayoutConstants.wideSecondaryThreshold` 已定义为 1180pt，但目前没有对应的宽屏次级栏，所以截图中的宽屏确实显得稀薄。
- Sidebar 已收敛为“工作站 / 设备”；连接和诊断都从 Toolbar 打开 Sheet。这一结构不可回退。
- `WorkspaceView` 已删除“管理工具”总入口，并定义“连接状态”为唯一连接目的地。第二轮不得重建工具管理页或第二个连接详情 Sheet。
- `DiagnosticsSheet` 已按“健康摘要 → 支持信息 → 原始日志”组织。原始日志是排障证据，不得为了干净而静默截断或改写。
- `RemoteAccessView` 已在单 Sheet 内承载 Relay、LAN 与高级展开；主要压力是纵向密度和滚动负担，不是缺少更多卡片。
- 评审时发现第二轮设计文档和四张参考截图原先被 `/docs/*` 忽略；既然它们是后续实现的可复核设计输入，已为这五个精确路径解除忽略。不得把整个 `docs/assets/` 目录改成无条件追踪，以免重新引入本地工作笔记和临时素材。

## 截图评估

### 工作站

“首屏稀薄”属实：健康结论、四个 backend 与 Relay 只占短文字柱，运行控制又漂在右上方，缺少稳定视觉锚点。

根因并不只是 880pt 上限。把短内容改成 960pt 居中仍会漂浮；每段套圆角卡又会得到三张小 dashboard 卡。可用横向空间应在宽屏展示不重复的只读“此 Mac”辅助信息（设备数量、最近连接、自动连接摘要），而不是放大空白或复制状态。

### 设备与配对

步骤轨迹、QR 与手动代码已有清晰顺序。问题是设备清单的密度和可点击性弱，不是配对区缺一个更大的“复制链接”按钮。QR 是标准路径的主视觉锚点；复制链接应维持替代动作。

### 帮助与诊断

摘要部分已经清楚。原始日志展开后占据注意力是合理的——它服务排障而非日常浏览。应提供最近片段与完整 200 行之间的明确切换，并明确日志为“原样证据”，而不是把它装饰性格式化。

### 连接状态

滚动负担来自 Relay 说明、开关、配置、LAN 详情和高级项同时堆叠。先压缩重复安全说明和默认 endpoint 展示，再隔离高级配置；更多边框不能减少同一份文字的高度。

## 逐项裁决

| 原方案建议 | 裁决 | 理由与约束 |
| --- | --- | --- |
| 工作站居中并放宽至 960–1000pt | **采纳，但改写** | 920–1179pt 单列；≥1180pt 主工作列 + 240–280pt 只读辅助栏。`wideSecondaryThreshold` 必须真正接入布局。工作列可试 900–960pt，但不能把无依据的固定数散落在多个 View。 |
| AI 工具与安全连接都使用轻量卡片 | **部分采纳** | 健康结论/注意事项可有一张承载状态和动作的交互面；AI 工具应为分组行，Relay 为紧凑状态行。拒绝每段、每行套卡，避免 dashboard-card mosaic。 |
| 查看设备、连接状态都升为大号蓝色主按钮 | **不采纳** | 首次使用的唯一主动作是“添加设备”；已配对且正常时无需蓝色主按钮。查看设备、连接状态保持次级，故障时才在对应行升级为修复 CTA。 |
| 后端每行增加图标、状态和右侧操作 | **部分采纳** | 名称、图标、状态、hover 合理；正常行不常驻“测试/管理”。失败行才按真实能力显示重新检查或连接状态。 |
| Relay 卡片并增加连接状态出口 | **部分采纳** | 可加强图标与状态层级，但仍只允许既有“连接状态”入口；不得新建连接详情目的地或双入口。 |
| 已授权设备做卡片列表 | **采纳，但改写** | 用一个分组容器中的原生列表行，不为每台设备单独制卡。行保留图标、名称、最近连接和 `…`；空状态给出添加设备动作。 |
| QR 区加浅色容器，复制链接升级主操作 | **部分采纳** | 可加极轻底色、增强当前步骤；复制链接维持次级，既有 `PairingView.uiState`、审批与过期语义不变。 |
| 连接 Sheet 用 `LabeledContent`、DisclosureGroup + 内部卡片 | **部分采纳** | `LabeledContent`/DisclosureGroup 可统一语义；高级内容仅一个 inset group，不为 Tailscale/自定义地址各加卡。默认隐藏 endpoint、重复 bullet 与无效保存按钮，优先降低滚动高度。 |
| 诊断三张健康卡；日志只显示 30 行或换 TextEditor | **部分采纳** | 健康摘要用三条统一状态行，不做窄 Sheet 中的三卡。可默认最近 30 行，但必须一键显示最近 200 行、原样复制；不合成行号/时间戳，也不以 `TextEditor` 取代当前只读等宽列表。 |
| 新建全局样式文件与通用卡片 | **部分采纳** | `AgentStatusRow` 有真实复用价值；只有健康/注意事项最终共享可点击面时才抽 `StatusSurface`。不预建大而全样式库，先从现有 `glassPanel` 和少量 token 提炼。 |
| 统一状态点、字体、辅助文案 | **采纳** | 建立最小 token 集：8pt 点、状态语义色、section 标题、辅助文字、行高、hover；状态必须同时用文字/图标表达。 |

## P0 实施规格

### 工作站自适应布局

- 以 `GeometryReader` 或 `ViewThatFits` 在 1180pt 分支：窄/常规窗口单列；宽窗口为主工作列和只读辅助栏。
- 辅助栏只显示非重复事实：Mac 名称、已授权设备数/最近连接、Relay 自动连接摘要；不得复制 backend 行，也不得承载设置或连接动作。
- 主列第一块是健康结论与当前唯一状态相关动作；注意事项仅在异常时显示。AI 工具与安全连接使用分隔线、间距和行层次。
- 首次使用仍以“添加设备”为唯一 `.borderedProminent` 大按钮；常态按钮不得与其竞争。

### 组件与三个 Sheet

- 可新增 `AgentStatusRow`，输入仅为现有 `BackendAgentStatus` 与可选故障动作；组件不得重新探测 backend 或制造假状态。
- 若健康结论与注意事项确实共享同种可点击表面，再抽取窄的 `StatusSurface`；否则保留局部布局。
- 设备清单改为分组面；撤销保持 `…` 后和现有确认语义。配对只调整 QR 焦点、步骤当前态、行距。
- 连接先删默认不必要的 endpoint/重复说明；Relay/LAN 为自动连接摘要的状态行，高级内容仅 DisclosureGroup 内出现。
- 诊断保留原始日志证据完整性；“30 行”只是浏览默认，绝不是采集或保留上限。现有复制成功反馈应使用短暂、原生且可访问的状态变化，而非悬浮 toast。

## 验收与验证边界

- 920pt 最小宽度下无横向截断，首要状态与唯一主动作无需滚动。
- ≥1180pt 宽度下，辅助栏提供非重复只读上下文，首屏不再是短文字柱悬浮在空画布上。
- 常态只有一个主操作；故障时才出现针对真实失败的行内修复动作。
- AI 工具、Relay、设备、诊断均以文字和图标表达状态；配对、撤销、Relay/Tailscale/自定义地址语义和 API 调用不变。
- 日志可完整查看最近 200 行并原样复制。
- 代码完成后运行 Swift 定向单测及 macOS Release build，并按项目约定覆盖安装到 `/Applications`、核对嵌入 runtime/端口/进程/日志。UI automation、snapshot、simulator 或真机 UI 验收仍需 owner 明确授权。

## 下一步

第二轮应从“加卡片”转为“有目的的布局和层级”。P0 先做工作站宽屏辅助栏、状态分组行和 CTA 层级；P1 收口设备/连接；P2 优化诊断浏览控制。

**不采纳项**已在“逐项裁决”中明确：双主 CTA、全区域卡片化、复制链接升为主要操作、以 `TextEditor` 替换原始日志。它们会破坏信息优先级、真实诊断证据或原生克制感。
