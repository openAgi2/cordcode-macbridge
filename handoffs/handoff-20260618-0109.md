## 3. Known Issues (遗留问题)

1. **【用户主诉·未修】iOS 反复 loadMessages → codex 长 session「循环 loading、消息页频繁跳动」**。pre-existing，被频繁重连掩盖。
   - 现象：codex 019ec076 的 get_session_messages 是 **newest/backward 1:1 振荡**（hasBeforeCursor false 64 次 + true 72 次 / 20min，~20-30s 一组），iPhone UI 频繁重新渲染。短 session 不触发。
   - 已排除的触发源：Mac cursor_stale（0 次）、Mac event/milestone 推送（无，Mac 日志 delivery/outbox/reconcile/milestone 全 0）、mailbox reconcile marker 残留（`completeMailboxReconcile` 在 fetchMessagesPage 成功后已清，CCCodeBridgeBackendClient.swift:293）。
   - 待排查：iOS loadMessages 有 **13 个调用点**——Generation.swift 5 个（:904,:1227,:1264,:1291,:1365）、CodexStreaming.swift:207（authoritativeReconcileRequired）、Todos:239、SessionManagement:417（initialize）/ :790（重连后）、MessageSync:410 / :480（cursor_stale）、ChatViewModel:302（重连后 resume）。需逐个加 NSLog 抓真机日志（iPhone 连 Xcode console），定位哪个在 ~20-30s 周期触发。重点怀疑 Generation（codex resume_session 是否误置 generating 状态）或 eventStream（CCCodeBridgeBackendClient:440）反复订阅。

2. **【次要·未修】Mac transcript replay fallback → claude 长 session「跳动」**。pre-existing。claude session 994f2d05 的 transcriptindex replay 只匹配 **75/77 条**（content replayer 漏 2 条），fallback full-parse 返回**全量 1.3MB**（response_bytes=1332724，绕过 maxPageResponseBytes）→ iOS 反复渲染跳动。codex 019ec076 **不**触发此路径。
   - 根治需改 `transcriptindex/index.go:263 PageIndex.Replay` 返回 ordinals，mismatch 时用已产出的部分 entries（75 条）走分页+cursor，不 fallback。障碍：`core/message.go:249 RichHistoryEntry` 无 ordinal 字段，`pagination.go:191` wire 映射靠 `pageSpans[i].Ordinal` 对齐，mismatch 时 i 错位。Replay 调用点单一致（pagination.go:180）；测试在 transcriptindex/transcriptindex_boundary_test.go:228,263 + transcriptindex_smoke_test.go:319,336（改签名要同步）。
